# Система приёма и анализа данных телескопа

Моделирует передачу данных от телескопа по протоколу UDP, их приём, хранение и предоставление через API для дальнейшего анализа на Python.

## Архитектура

1. **Телескоп** (симулятор) – отправляет UDP-пакеты с кадрами.
2. **Go-сервер** – слушает UDP-порт, собирает пакеты в кадры, проверяет целостность, сохраняет в памяти и предоставляет HTTP API.
3. **Python-клиент** – запрашивает данные через API, обрабатывает с помощью NumPy/SciPy, строит графики, ищет аномалии.

## Формат UDP-пакета

| Поле         | Размер (байт) | Тип     | Описание                     |
|--------------|---------------|---------|------------------------------|
| FrameID      | 4             | uint32  | Уникальный номер кадра       |
| PacketIndex  | 4             | uint32  | Индекс пакета внутри кадра   |
| TotalPackets | 4             | uint32  | Всего пакетов в кадре        |
| Timestamp    | 8             | uint64  | Микросекунды с эпохи         |
| DataLength   | 2             | uint16  | Длина полезной нагрузки (байт)|
| Data         | переменная    | []byte  | Сырые данные (например, float32)|

Заголовок фиксированный – 22 байта.

## Быстрый старт

### Установка Go-сервера

```bash
git clone <ваш-репозиторий>
cd go-udp-server
go mod tidy
```

### Запуск сервера

```bash
go run ./cmd/server
```

Сервер будет слушать:

- UDP на порту 9000 (можно изменить в коде)

- HTTP API на порту 8080

### Запуск симулятора телескопа

В отдельном терминале:

```bash
go run ./cmd/simulator
```

### Запуск Python-клиента

Установите зависимости:
```bash
pip install requests numpy matplotlib scipy
```

Запустите:

```bash
python client.py
```

### API эндпоинты

GET /frames?from={id}&to={id} – получить все кадры в диапазоне

GET /frame/{id} – получить конкретный кадр

Данные возвращаются в JSON, поле data закодировано в base64.

Пример запроса:

```bash
curl "http://localhost:8080/frames?from=1&to=10"
```

## Расширение функциональности

- Для постоянного хранения замените storage.go на работу с PostgreSQL / TimescaleDB.

- Для работы с реальным телескопом адаптируйте формат пакетов и адрес назначения.

- Для производственной среды добавьте graceful shutdown, Prometheus метрики и конфигурацию через переменные окружения.
